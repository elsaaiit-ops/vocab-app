# 單字練習 PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline). This is a no-build vanilla project; instead of TDD steps, each task has explicit manual verification steps (browser + console).

**Goal:** Build a cross-device-sync vocabulary PWA per `vocab-app-build-spec-v2.md`, deployable to GitHub Pages with Supabase backend, offline-capable on mobile.

**Architecture:** Pure static HTML/CSS/ES-module JS. Local-first: every read hits localStorage immediately, every write goes to `pendingOps` queue then background-syncs to Supabase Postgres via `@supabase/supabase-js@2` loaded from esm.sh. Conflict resolution = last-write-wins on `updated_at`. Service Worker caches app shell for offline.

**Tech Stack:** Vanilla HTML/CSS/JS (ES modules), Supabase JS via CDN, Service Worker, Free Dictionary API.

## Global Constraints

- No framework, no build step, no npm install. Only ES modules.
- Supabase via esm.sh: `import { createClient } from "https://esm.sh/@supabase/supabase-js@2"`.
- ALL paths relative. `manifest.start_url` = `"./"`, `manifest.scope` = `"./"`, SW registered with `"./sw.js"`.
- DB and JS both use **snake_case** (no translation).
- Settings live in `config.js` (`SUPABASE_URL`, `SUPABASE_ANON_KEY`). anon key public is OK — RLS protects rows.
- Every write sets `updated_at = Date.now()`.
- Deletes are soft (`deleted = true`) + upsert. UI filters out `deleted === true`.
- Mobile-first; buttons ≥ 44px high. Traditional Chinese UI text.
- localStorage key: `"vocab-app"` containing `{ words, pendingOps, settings }`.

### Task 3: srs.js — Leitner

**Files:**
- Create: `srs.js`

**Interfaces:**
- Produces:
  - `INTERVALS = [null, 0, 1, 3, 7, 14]` (1-indexed for box)
  - `isDue(w, now = Date.now())` → boolean
  - `gradeCorrect(w)` → mutated copy with `box = min(box+1, 5)`, `last_reviewed`, `seen++`, `correct++`, `updated_at`
  - `gradeWrong(w)` → mutated copy with `box = 1`, `last_reviewed`, `seen++`, `updated_at`
  - `sortDue(words)` → returns due-only, sorted by `box` asc, `last_reviewed` asc (nulls first), then Fisher-Yates within same `(box, last_reviewed)` bucket.

**Steps:**

- [ ] Implement constants and pure functions.
- [ ] No I/O, no DOM — just data in / data out.

**Verify:** In browser console: 
```js
const m = await import('./srs.js');
const w = {box:1,last_reviewed:null,seen:0,correct:0,deleted:false};
m.isDue(w) === true;           // new word — due
m.gradeCorrect(w).box === 2;   // box up
m.gradeWrong({...w,box:4}).box === 1;  // back to 1
```

### Task 4: store.js — local cache + pendingOps + sync()

**Files:**
- Create: `store.js`

**Interfaces:**
- Consumes: `supabase.js`
- Produces:
  - `loadState()` → `{ words, pendingOps, settings }` (from `localStorage["vocab-app"]`, defaults seeded)
  - `saveState(state)` → write back to localStorage
  - `addWord(partial)` → returns full Word with id (uuid v4 via `crypto.randomUUID()`), `box:1`, `deleted:false`, `updated_at:Date.now()`, etc. Pushes onto words + pendingOps. Triggers `sync()`. Returns `{ word, dedupedTo }` — if word already exists case-insensitive on `word`, returns existing without insert.
  - `gradeWord(id, correctBool)` → updates word via srs, pendingOps, sync.
  - `deleteWord(id)` → soft delete, pendingOps, sync.
  - `getWords()` → from cached state, filtered `deleted === false`.
  - `getAllIncludingDeleted()` → for sync diffs.
  - `sync()` → push pendingOps via `pushUpserts`, then `pullAll`, merge by id (`updated_at` larger wins), persist, fire `'vocab:changed'` event on `window`.
  - `getSyncStatus()` → `{ pending: number, online: boolean, lastSyncAt: number|null }`.
  - `installSyncTriggers()` → wires `window.online`, `window.focus`, 30s interval.

**Steps:**

- [ ] All localStorage I/O goes through one `read()`/`write()` helper that does JSON parse/stringify with try/catch.
- [ ] `sync()` checks `!navigator.onLine` or `!user` and returns.
- [ ] After write, dispatch `new CustomEvent('vocab:changed')` so UI re-renders.
- [ ] Use `crypto.randomUUID()` for ids.
- [ ] Dedup check: lowercase `.trim()` comparison on `word`.

**Verify:** Console:
```js
const s = await import('./store.js');
const r = s.addWord({word:'hello', meaning:'你好'});
s.getWords().length >= 1;
s.getSyncStatus().pending >= 1;   // queued because not signed in
```

### Task 6: 新增 tab — 3 sources (manual / dictionary / JSON list)

**Files:**
- Modify: `app.js` — implement `renderAddTab()`

**Steps:**

- [ ] Three sub-forms inside the tab; toggled via radio or sub-buttons.
- [ ] **Manual:** `word` (required), `meaning` (required), `example` (optional). Submit calls `addWord({...form, source:'manual'})`. On dedup-hit, show "已存在,跳過".
- [ ] **論文 / Dictionary:** single `word` input. On submit, fetch `https://api.dictionaryapi.dev/api/v2/entries/en/{word}`. Populate editable preview with `part_of_speech` (first meaning's `partOfSpeech`), `meaning` (first definition), `example` (first definition's `example`). User edits then confirms → `addWord({..., source:'paper'})`. On fetch error / non-200 → fallback to manual form, prefilled with the word and an error banner "API 失敗,請手動填入".
- [ ] **字表匯入:** textarea for JSON paste + file input. Parse `[{word, meaning, example?}]`. For each entry call `addWord({..., source:'list'})`. Show summary "匯入 X 筆,跳過 Y 筆(已存在)".
- [ ] Provide a "範例" button populating textarea with 12-entry starter list.

**Verify:** Add one manual word → My words tab shows it. Use dictionary fetch on "ubiquitous" → preview appears. Paste JSON of 3 entries → all 3 saved.

### Task 8: 練習 tab — flashcard + spelling

**Files:**
- Modify: `app.js` — `renderPracticeTab()`

**Steps:**

- [ ] Build due queue once via `sortDue(getWords())`. If empty → render "今天沒有要複習的字 🎉".
- [ ] Two mode buttons at top: 字卡 / 拼寫.
- [ ] **Flashcard:** show `word`. Click card → flip to `meaning + example`. Bottom: 我會✓ / 我不會✗ buttons. On click → `gradeWord(id, bool)` then advance index. Progress `n/m` displayed.
- [ ] **Spelling:** show `meaning` + `(part_of_speech)`. Input + Enter or 提交 button. Compare: `input.trim().toLowerCase() === word.trim().toLowerCase()` → correct. Else compute Levenshtein, if `≤1` and `settings.typoTolerance` → "差一點點,正確: {word}" and grade as wrong. Else "錯了,正確: {word}" wrong. Advance.
- [ ] After last card → "完成 X 題" + 重新練習 button (rebuilds queue).

**Verify:** Add 3 due words, enter practice, mark correct → box up; mark wrong → box back to 1. Spelling: typo "helloo" vs "hello" triggers tolerance branch.
